# Next/Insight 画面 再設計（回復期UX）

## Before / After Widget 構造

### Before

```
Scaffold
└── SafeArea
    └── SingleChildScrollView (padding: 24h, 20v)
        └── Column (mainAxisSize: min)
            ├── SizedBox(104)
            │   └── WaveHeader (useFixedHeightLayout: true)
            │       └── Stack
            │           ├── Align(topCenter) > SparklineWave
            │           └── Positioned(right: 16, bottom: 8) > Text  ← 右寄せ
            ├── SizedBox(28)
            ├── _TodayBlock (体力・集中・疲れ)
            ├── SizedBox(40)
            └── _HistoryBlock
```

- 波とラベルが **Stack 内でラベル右下固定**。視覚的に右に寄り、装飾のように見える。
- 波と入力UIが同じ ScrollView 内で並ぶだけなので、「波＝上位概念」が伝わりにくい。

### After

```
Scaffold
└── SafeArea
    └── Column
        ├── SizedBox(120)                    ← 固定高さヘッダー領域
        │   └── _WaveHeaderSection
        │       └── Column (mainAxisAlignment: center)
        │           ├── Center > SparklineWave
        │           ├── SizedBox(6)
        │           ├── InsightMessage（観測の解釈・1行〜2行）
        │           ├── SizedBox(4)
        │           └── Text("今日のリズム：ゆるやか", textAlign: center)
        ├── SizedBox(32)                     ← 明確な区切り
        └── Expanded
            └── SingleChildScrollView
                └── Column
                    ├── _TodayBlock
                    ├── SizedBox(40)
                    └── _HistoryBlock
```

- 波＋ラベルは **Column で中央揃え**。Positioned(right:) なし・Stack で重ねなし。
- 「Wave Header（120px）」→「32px 余白」→「入力エリア」の **縦の階層** がはっきりする。

---

## 修正理由

1. **意味の階層をはっきりさせる**  
   波を「今日全体の状態」として画面上部の固定ヘッダーにし、その下に体力・集中・疲れを置くことで、「今日のリズム → その内訳」という順序が伝わる。

2. **右寄せラベルをやめる**  
   `Positioned(right: 16, bottom: 8)` を廃止し、`textAlign: TextAlign.center` に統一。波とラベルを一つのブロックとして中央に置き、装飾ではなく「見出し」として認識されやすくする。

3. **レイアウトを Column + Expanded に統一**  
   Stack で重ねず、Column で上から「Wave Header → 余白 → 入力エリア」と並べる。スクロールは入力エリアのみに限定し、ヘッダーは常に同じ位置で「今日のリズム」として固定する。

4. **ラベル文言を「今日のリズム」にそろえる**  
   「波：ゆるやか」→「今日のリズム：ゆるやか」、「推定リズム：N日」→「今日のリズム：N日」に変更し、「今日」の文脈を一言で示す。

## 回復期UX観点での改善点

| 観点 | 内容 |
|------|------|
| **0.3秒で文脈が分かる** | 画面上部に「今日のリズム」が中央で固定され、スクロールしなくても「今日はこういう波の日」と理解しやすい。 |
| **認知負荷の軽減** | 波とラベルが一か所にまとまり、右寄せや装飾的な配置をやめたことで、「何がメインか」が迷わず分かる。 |
| **階層の明確化** | ヘッダー（波）→ 入力エリア（体力・集中・疲れ・履歴）の2段構成にし、回復期ユーザーが「まず全体、次に詳細」と追いやすい。 |
| **一貫した操作領域** | スクロールは入力エリアだけに限定し、ヘッダーは触れずに「見るだけ」の領域にすることで、操作範囲の予測がしやすい。 |

## WaveHeader の 3 層構成（AIの一言）

WaveHeader は **波（SparklineWave）＋ 解釈（InsightMessage）＋ ラベル（今日のリズム）** の 3 層で構成する。

- **解釈** は直近の waveScore から生成した短文（1〜2行）。「上向き/下向き/安定」の傾向と、必要に応じて振れ幅の補足を表示する。
- entries が 3 件未満のときは解釈は表示しない（波＋ラベルのみ）。

### 命令しない観測文にした理由

回復期ユーザーにとって、アドバイスや断定は負荷になりうる。そのため解釈文は **観測の事実** に限定し、次のようにしている。

- **命令・アドバイス禁止**: 「〜しよう」「〜してください」は使わない。
- **断定を避ける**: 「〜です」より「〜のようです」「〜かもしれません」を用いる。
- **医療語禁止**: 診断・治療・症状・改善などの表現は使わない。

その結果、波が「装飾」ではなく「観測結果の要約」として伝わり、0.3 秒で文脈を掴みやすくする。

### 文言ガイドライン

| 推奨 | 禁止 |
|------|------|
| 推定表現（「〜の傾向かもしれません」「おおむね〜」） | アドバイス・命令（「〜しよう」「〜すべき」） |
| 観測の事実（「ここ数日は〜」「振れは小さめ」） | 医療用語（診断・治療・症状・改善など） |
| 天気予報的なトーン | 断定・ネガティブの強調 |
